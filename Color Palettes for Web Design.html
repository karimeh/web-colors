🎨 Color Palettes for Web Design

Definitions, visual examples, pros/cons, and how to define them in CSS

A color palette is a set of colors used together in a design. Choosing the right palette type can enhance aesthetics, usability, and brand identity. Below you'll find the most common palette types, each with a brief explanation, live swatches, and a breakdown of advantages and disadvantages.

All examples use standard CSS color definitions – you can copy the hex codes directly into your stylesheets.

Monochromatic

Uses a single base hue and extends it with its tints (add white), shades (add black), and tones (add gray). Creates a clean, cohesive look.

📐 How to Define Colors in CSS

All palettes above can be implemented using any of these methods. Here are the most common ways to define colors for the web:

  • Named Colors: color: red; (147 named colors, e.g., rebeccapurple)
  • HEX: background: #ff5733; (short #f00, full #ff0000, or 8‑digit with alpha #ff000080)
  • RGB / RGBA: color: rgb(255, 0, 0); / rgba(255,0,0,0.5); (modern space‑separated: rgb(255 0 0 / 0.5))
  • HSL / HSLA: background: hsl(200, 100%, 50%); (intuitive for palette creation: change hue for variations)
  • HWB: border-color: hwb(0 0% 0%);
  • Lab / LCH / Oklab / Oklch: background: oklch(0.7 0.2 250); (perceptually uniform, great for gradients)
  • Custom Properties (Variables): --brand: #6366f1; background: var(--brand); (reusable, dynamic)
  • color-mix(): background: color-mix(in srgb, var(--brand) 60%, white); (create tints/shades programmatically)
  • Relative Color Syntax: background: hsl(from var(--brand) h s calc(l + 20)); (lighten/darken a color)

💡 For palette harmony, using HSL is often the easiest – just rotate the hue while keeping saturation and lightness consistent.
